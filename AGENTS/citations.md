# Citations

Design systems and headless component libraries that Lily learns from. Each is a useful reference when shaping component APIs, naming, accessibility patterns, and visual reference.

## Headless / unstyled component libraries

- [Ark UI](https://ark-ui.com/) — multi-framework headless components from the Chakra team
- [Bits UI](https://bits-ui.com/) — Svelte headless component library
- [Melt UI](https://www.melt-ui.com/) — Svelte headless builders
- [Radix UI](https://www.radix-ui.com/) — React headless primitives
- [shadcn/ui](https://ui.shadcn.com/) — copy-paste React components on top of Radix
- [Reuters Graphics components](https://github.com/reuters-graphics/graphics-components) — Svelte components for editorial graphics

## Styled / opinionated design systems

- [Adobe Spectrum](https://spectrum.adobe.com/page/code/) — Adobe's design system
- [Ant Design](https://ant.design/) — React component framework
- [Carbon Design System](https://carbondesignsystem.com/) — IBM
- [DaisyUI](https://daisyui.com/) — Tailwind component library
- [Flowbite](https://flowbite.com/) — Tailwind UI components
- [Mozilla Protocol Design System](https://protocol.mozilla.org/) — Mozilla
- [Skeleton](https://www.skeleton.dev/) — Svelte + Tailwind UI toolkit
- [Wonderflow Wanda](https://design.wonderflow.ai/) — Wonderflow
- [Design System AU](https://designsystemau.org/) — Australian Government

## Public-sector reference designs (the visual languages Lily's example apps target)

- [NHS UK Design System](https://service-manual.nhs.uk/design-system) — current default reference visual language
- [GOV.UK Design System](https://design-system.service.gov.uk/)
- [ONSdigital Design System](https://github.com/ONSdigital/design-system) — UK Office for National Statistics
- [U.S. Web Design System (USWDS)](https://designsystem.digital.gov/) — maintained by 18F / GSA
- [18F UI Library](https://designsystem.digital.gov/) — historically led USWDS development
